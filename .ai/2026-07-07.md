### 2. Design understanding
- Repo is **spec-first**: `spec/` has 9 detailed modules (~9.5k lines); code is mostly stubs.
  `packages/engine/src/index.ts` = typed skeleton, `resolveTurn` throws "not implemented".
- The game is **Battlesnake + teams + potions**. Base layer (simultaneous moves, wall/self/body
  collisions, head-to-head longer-wins, food/health, hazards) ~= Battlesnake 1:1. Novel layer:
  teams, the potion **buff/debuff family-rebuild** mechanic (the pedagogical core), severing via
  invulnerability levels, invisibility, chess timer, deterministic seeding.
- Architecture = 3 runtimes: **SpacetimeDB** (authoritative turn resolution, per-game),
  **Convex** (accounts/rooms/replays, persistent), **Centaur Servers** (per-team bot + operator UI).
- Cadence: **live, synchronous, blitz-chess-paced** single sitting (chess clock: 60s budget,
  +500ms/turn, 10s/turn cap, ~100 turns) — NOT correspondence over days. This makes multiplayer
  the *easy* lockstep kind (submit-move -> wait for all/deadline -> resolve -> broadcast).

### 3. Built a reduced v0 playable demo
Strategy: skip the whole 3-runtime platform; build the pure engine + a board in one vite process.
- **`apps/centaur-server-reference/src/lib/snek/engine.ts`** — reduced `resolveTurn`
  (Move -> Movement -> Collisions[wall/self/body/head-to-head] -> Health/Food -> Food-spawn -> Win),
  plus `createInitialState`, mulberry32 deterministic RNG, and a greedy `computeBotMoves` bot.
  Types mirror `@cyphid/snek-engine` so it lifts into `packages/engine` later. Phase comments
  cite `01-REQ-*`. Collision rules are already spec-correct for the level-0 (no-potion) case.
- **`apps/centaur-server-reference/src/routes/demo/+page.svelte`** — Battlesnake-style SVG board,
  Step/Play/Reset/speed, team stats, event log, win banner. Svelte 5 runes. Live at **/demo**.
- Home page (`src/routes/+page.svelte`) got a link to `/demo`.

### 4. Bug found + fixed (browser-verified)
- Symptom: pressing Play didn't animate.
- Root cause: engine calls `structuredClone()` on the state it's given; the UI passed Svelte 5's
  reactive **$state proxy**, and `structuredClone` can't clone a Proxy -> `DataCloneError` thrown
  inside `step()` before `state = next` ran. Headless test missed it (Node cloned plain objects).
- Fix: `const snapshot = $state.snapshot(state)` before calling the engine. Verified in-browser:
  turn counter advances, snakes move, console clean.

## Gotchas / notes for next session
- **Repo is on `/mnt/c` (drvfs) -> no inotify -> Vite HMR does NOT fire.** Every code change or new
  route needs a full dev-server restart (`fuser -k 5000/tcp` then relaunch; ~30s to ready).
  Consider a Linux-filesystem clone (`~/prog/`) for instant HMR when this becomes sustained work.
- Detached restart under WSL is finicky — `nohup setsid ... &` from a persistent invocation is more
  reliable than a one-shot `wsl.exe` that returns immediately.
- The demo engine lives in the **app** (`src/lib/snek/`), not `packages/engine`, for zero-friction
  demoing. It's copy-compatible; migrate when wiring the real engine.

### Adversarial review (independent agent, headless repros against spec 01)
- **No correctness deviations found in the implemented subset.** Verified: head-swaps -> both die
  BodyCollision; head-to-head on food -> longer survives and eats, ties die and food remains;
  eat-restores-health-before-starvation-check (01-REQ-046); growth timing (01-REQ-062);
  dead bodies stop blocking; tail-chase safe unless leader ate; stacked start clean;
  collision passes are order-independent (single post-move snapshot + death Set).
- **MED**: turn-limit tiebreak uses raw aggregate length, not 01-REQ-053 normalized body-share.
  Identical winner for 2 teams (same denominator); latent bug for 3+ teams. Documented, not fixed.
- **MED (cosmetic)**: greedy bots don't coordinate — ~31% of random seeds contain a same-team
  mutual head-to-head that wipes a team. Engine-correct but looks broken. Default **seed 12345 is a
  clean run** (Red wins turn 32); avoid random Reset when demoing, or add teammate-awareness.
- LOW: turn-0 "Up" fallback is dormant (bot stages every move); SnakeMoved lacks `stagedBy`
  (01-REQ-052); double deep-copy per turn costs 61–277 µs even at 32x32/20 snakes (non-issue).
